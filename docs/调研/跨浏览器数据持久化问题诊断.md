# 跨浏览器数据不持久化问题诊断

> 诊断日期：2026-05-04
> 问题：不同浏览器登录相同账号，用户偏好和设置全部丢失，需重新填写

---

## 一、根因分析

前端大量用户偏好和状态存储在 `localStorage` 中，而 `localStorage` 是浏览器隔离的——换个浏览器就无法读取。

后端的 `users` 表已有 `profile_data`（JSON 字段）可以承载这些偏好，但**目前未被利用**。

---

## 二、存储分布对比

### 服务端存储（跨浏览器正常）

| 数据 | 存储位置 | 说明 |
|------|---------|------|
| 用户认证 | HttpOnly Cookie | 会话 token，自动携带 |
| 模型配置 | `model_configurations` 表 | API Key、端点、选中模型 |
| 会话列表 | `conversations` 表 | `/api/conversations` 接口 |
| 聊天历史 | `conversation_records` 表 | `/api/chat/history/{id}` 接口 |
| 用户画像 | `users` 表 | 昵称、头像、邮箱、电话 |
| 定时任务 | `scheduled_tasks` 表 | `/api/scheduled-tasks` 接口 |
| 插件/技能 | `plugins`/`skills` 表 | 服务端全量管理 |

### localStorage 存储（跨浏览器丢失）

| Key | 内容 | 读写文件 | 新浏览器表现 |
|-----|------|---------|------------|
| `theme` | 深色/浅色主题 | `themeStore.ts` | 重置为浅色 |
| `app_settings` | 主题/语言/apiProvider/确认模式/审计/工具轮次 | `SettingsPage.tsx` | 全部默认值 |
| `chat_selected_model` | 当前选中模型 | `chatStore.ts:194` | 清空 |
| `chat_thinking_enabled` | 思考模式开关 | `chatStore.ts:209` | 关闭 |
| `chat_thinking_depth` | 思考深度 0-5 | `chatStore.ts:214` | 重置为 0 |
| `chat_output_mode` | 流式/非流式 | `chatStore.ts:189` | 重置为 stream |
| `chat_cache_v1` | 活跃会话 ID + 消息缓存 + 会话摘要缓存 | `chatCache.ts` | 全部清空 |
| `log_level` | 日志级别 | `logger.ts:29` | 重置为默认 |
| 定时任务模板 | 自定义模板 | `TaskTemplateManager.tsx:26` | 清空 |

### 纯内存存储（刷新即丢失，但 /auth/me 会恢复）

| 状态 | 文件 | 说明 |
|------|------|------|
| `useAuthStore` | `authStore.ts` | user/token/isAuthenticated — 无 persist 中间件 |

---

## 三、影响评估

### 用户感知

- **设置页**：主题、语言、默认 Provider、确认模式等全部恢复默认，需要重新配置
- **聊天页**：选中的模型丢失、思考模式关闭、输出模式回退
- **会话列表**：虽然服务端有数据，但活跃会话 ID 丢失，需手动选择
- **定时任务模板**：自定义模板全部丢失

### 不受影响

- 模型 API Key 配置（DB 存储，登录后可见）
- 历史聊天记录（服务端拉取）
- 插件/技能安装状态

---

## 四、修复方向

### 后端

`users` 表已有 `profile_data JSON` 字段，可以直接存储偏好：

```python
# 新增接口: GET/PUT /api/user/preferences
# 读写 users.profile_data["preferences"] 子对象
```

偏好数据结构示例：

```json
{
  "preferences": {
    "theme": "dark",
    "language": "zh-CN",
    "api_provider": "openai",
    "selected_model": "deepseek-chat",
    "thinking_enabled": true,
    "thinking_depth": 3,
    "output_mode": "stream",
    "require_confirm": true,
    "enable_audit": false,
    "max_tool_call_rounds": 5
  }
}
```

### 前端

1. 在 `userAPI` 中新增 `getPreferences()` / `updatePreferences()` 方法
2. 各 Store（`themeStore`、`chatStore`）在初始化时从服务端拉取偏好
3. 偏好变更时同步写入服务端 + 更新 localStorage 本地缓存
4. localStorage 降级为离线缓存，服务端为权威数据源

### 不需要迁移的

- `chat_cache_v1`（消息缓存）— 这是纯性能缓存，跨浏览器不需要同步，新浏览器从服务端拉历史即可
- `log_level` — 开发者调试用，本地存储合理
- 定时任务模板 — 可考虑迁移，优先级较低
