# Open-AwA 代码健壮性与安全性复核报告

**复核日期**: 2026-04-17  
**复核范围**: backend/、frontend/src/、关键配置、回归测试  
**说明**: 本报告替代同名旧版扫描报告，仅保留本轮已重新核实的问题、已实施修复和最终验证结果。

---

## 一、执行摘要

本轮工作不是简单静态扫描，而是按“源码复核 -> 复现实验 -> 修复 -> 回归验证”的方式完成。旧报告中不少条目属于候选风险或历史状态，已不再直接采用。

本轮确认并处理的重点问题包括：

1. 本地账号初始化仍允许使用提交到仓库中的明文或占位密码。
2. 后端日志脱敏字段覆盖不完整，认证票据与会话标识存在泄漏风险。
3. 聊天上传文件缺少足够的归属和文件名约束，存在越权访问风险。
4. WebSocket 复用历史会话时缺少足够的归属校验。
5. 前端设置页将过宽的配置对象写入浏览器持久化存储。
6. 错误边界和侧边栏在异常展示、可访问性和测试稳定性上存在不足。
7. 若干回归测试本身存在隔离缺陷，导致全量运行时出现顺序相关失败。

本轮完成后，后端全量测试、前端全量测试和前端构建均已通过。

---

## 二、复核方法

本轮采用以下方法交叉确认问题真伪：

1. 逐文件阅读关键后端路由、配置、日志、模型服务和前端页面/组件实现。
2. 对候选问题执行定向复现实验，确认是否能在当前代码中真实触发。
3. 对已修复问题补充或修正回归测试，避免只修表象。
4. 进行后端全量 pytest、前端全量 Vitest 和前端生产构建验证。

## 六、剩余风险与后续建议

本轮仍观察到 1 个已复现但未在本次直接修改的剩余问题：

### R-01 Windows 下 Loguru 文件轮转冲突

- 现象: 后端全量测试期间，日志文件轮转偶发出现 `PermissionError: [WinError 32]`，表现为当前日志文件在重命名归档时被其他句柄占用。
- 影响: 不影响本轮业务测试通过，但会污染测试输出，并说明当前文件轮转策略在 Windows 场景下稳定性一般。
- 当前结论: 该问题更适合作为独立日志兼容性修复处理，避免为了消除测试噪音而引入新的日志行为回归。
- 建议方向:
  1. 为测试环境使用独立日志目录或禁用尺寸轮转。
  2. 评估将尺寸轮转改为时间轮转，降低高频 rename 冲突概率。
  3. 若需要继续保留当前策略，补充 Windows 专项验证。
