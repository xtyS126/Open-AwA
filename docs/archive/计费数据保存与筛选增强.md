# 计费数据保存与筛选增强计划

## 需求概述

1. 在计费页面的时间筛选下拉框中添加"全部"选项
2. 确保所有计费数据都被保存
3. 在设置页面添加最大保存天数配置

## 实现步骤

### 阶段1：后端修改

#### 1.1 修改 reporter.py - 支持"全部"时间筛选
- 修改 `get_cost_statistics()` 方法，当 period="all" 时返回所有历史数据
- 修改 `_get_period_dates()` 方法支持特殊情况

#### 1.2 修改 tracker.py - 支持最大保存天数
- 添加配置表 `retention_config` 用于存储保留策略
- 修改 `get_usage_records()` 方法，根据保留天数过滤数据
- 添加 `cleanup_old_records()` 方法用于清理过期数据

#### 1.3 修改 billing/routers/billing.py
- 添加 `GET /api/billing/retention` - 获取保留配置
- 添加 `PUT /api/billing/retention` - 更新保留配置

#### 1.4 修改 config/settings.py
- 添加 `USAGE_RETENTION_DAYS` 配置项（默认365天）

### 阶段2：前端修改

#### 2.1 修改 BillingPage.tsx
- 将 period 类型改为：`'daily' | 'weekly' | 'monthly' | 'yearly' | 'all'`
- 在下拉框中添加"全部"选项
- 修改 API 调用逻辑

#### 2.2 修改 SettingsPage.tsx
- 添加"数据保留"标签页
- 添加最大保存天数输入框
- 添加保存按钮和提示信息

#### 2.3 修改 billingApi.ts
- 添加 `getRetention()` 方法
- 添加 `updateRetention()` 方法

### 阶段3：测试与验证

#### 3.1 后端测试
- 测试"全部"时间筛选返回所有数据
- 测试保留天数配置保存
- 测试清理过期数据功能

#### 3.2 前端测试
- 测试"全部"选项显示
- 测试设置页面配置保存
- 测试数据按保留天数过滤
