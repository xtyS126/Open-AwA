# 微信自动回复模块 (Weixin Auto Reply) 更新日志

## [v2.0.0] - 2026-05-01

### 新增特性 (Features)

- **完整后台轮询引擎**：新增 `WeixinAutoReplyService` 提供高可用的后台独立轮询服务，支持 `start`、`stop`、`restart` 和运行状态监控。
- **自定义规则引擎优先匹配**：新增基于关键词和正则表达式的规则引擎（`WeixinAutoReplyRule`）。若用户消息匹配规则，直接回复预设内容，未命中规则时再降级调用大模型 (AI Agent)。
- **持久化幂等记录**：在状态文件中持久化记录近 500 条消息的处理状态（成功、失败、跳过），保证相同的 `message_id` 不会因为网络或服务重启而被重复回复。
- **消息可靠拉取与游标持久化**：重构了获取消息游标（`cursor`）的保存机制。只有在整批消息完全处理成功（无未捕获异常）后，游标才会进行更新并落盘，防止“先移游标后处理失败”导致消息丢失。

### 重构与优化 (Refactoring & Optimization)

- **AI 思维链 (Reasoning) 清洗隔离**：新增 `sanitize_weixin_reply_text`，对大模型生成的文本进行强力清洗，自动剔除 `<think>` / `reasoning` 包裹块及内部思考过程，确保向微信终端下发的文本仅包含最终回复。
- **并发锁与线程安全强化**：在 `WeixinSkillAdapter` 中针对 Windows 系统下的文件读写引入进程内共享读写锁 (`threading.RLock`)，并在并发写盘失败时增加指数退避重试（Exponential Backoff），大幅降低 `PermissionError` 的发生概率。
- **统一的消息提取模型**：实现了健壮的入站消息解析（`extract_weixin_text` 和 `normalize_inbound_message`），支持平铺结构、嵌套 `msg` 结构及 `item_list` 格式的内容抽取。
- **稳定消息 ID 算法**：为未提供主键的消息引入确定性 Hash 指纹计算（`build_weixin_message_id`），涵盖发送方、上下文 Token、内容文本及时间戳，确保幂等防重的精准性。
- **长文本安全截断机制**：对发送给微信的最终回复文本实行 Unicode 字符级截断（默认限制 1000 字符），防止超长消息导致上游接口拦截或拒绝服务。

### 缺陷修复 (Bug Fixes)

- 修复了因为自动回复拉取游标文件与运行时状态文件存放在一起，导致相互覆盖的潜在冲突。状态与游标文件已完全隔离。
- 修复了 AI 输出携带“最终答案：”等硬编码前缀直接推送到微信前端的问题。

---

*注：本次重构在不影响原有微信底层接口（`WeixinSkillAdapter`）适配能力的前提下，增强了自动回复闭环的鲁棒性与体验。*
