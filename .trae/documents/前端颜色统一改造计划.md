# 前端颜色统一改造计划

## 目标
- 将前端颜色使用统一到全局设计变量体系，减少硬编码颜色。
- 统一主要页面（仪表盘、计费、聊天、设置、技能弹窗、插件调试等）的视觉风格。
- 在不改变业务功能的前提下完成样式改造，并确保类型检查通过。

## 现状结论
- 项目已存在全局颜色变量定义，位于 `frontend/src/styles/global.css`。
- 多个页面已部分使用 `var(--color-*)`，但仍存在大量硬编码（如 `#333`、`#666`、`#E5E5E5`、`#5B8DEF`、`#dc2626` 等）。
- 部分 TSX 内联样式和图表颜色也直接写死，导致主题一致性不足。

## 统一规则
1. 优先使用全局变量：`--color-primary`、`--color-text`、`--color-border` 等。
2. 对通用语义颜色补齐变量：错误、警告、成功、遮罩、图表线条、图表网格。
3. 能用全局类与变量表达的颜色，不保留局部硬编码。
4. 品牌色（如 provider badge）若需保留差异，采用语义化变量映射，避免散落硬编码。
5. TSX 中内联颜色全部改为变量引用（CSS 类或 `var(--...)`）。

## 改造范围
- `frontend/src/styles/global.css`
- `frontend/src/pages/BillingPage.css`
- `frontend/src/pages/ChatPage.css`
- `frontend/src/pages/ExperiencePage.css`
- `frontend/src/components/SkillModal.css`
- `frontend/src/components/PluginDebugPanel.css`
- `frontend/src/pages/PluginsPage.css`
- `frontend/src/pages/SettingsPage.tsx`（内联颜色）
- `frontend/src/App.tsx`（初始化 loading 内联颜色）
- `frontend/src/pages/DashboardPage.tsx`、`frontend/src/pages/BillingPage.tsx`（图表颜色常量）

## 实施步骤
### 第一步：扩展全局颜色令牌
- 在 `global.css` 中新增并整理语义变量：
  - 状态色：`--color-success-bg`、`--color-error-bg`、`--color-warning-bg`
  - 图表色：`--color-chart-grid`、`--color-chart-axis`、`--color-chart-primary`、`--color-chart-secondary`
  - 遮罩与深色面板：`--color-overlay`、`--color-surface-dark`、`--color-text-on-dark`
  - 交互色：`--color-danger`、`--color-danger-hover`
- 保持现有变量兼容，避免一次性破坏已有样式。

### 第二步：统一页面样式文件中的硬编码
- 分文件替换常见硬编码到变量：
  - 中性色：`#333/#666/#999/#E5E5E5/#F5F5F5/#FAFAFA`
  - 主色系：`#5B8DEF` 及 hover 色
  - 状态色：成功/警告/错误相关色值
- 对 `BillingPage.css`、`SkillModal.css`、`ExperiencePage.css` 这类硬编码集中区域优先处理。

### 第三步：统一组件样式中的 fallback 颜色
- 清理 `var(--token, #fallback)` 中不必要 fallback。
- 若必须保留 fallback，仅保留与设计令牌一致的后备值。

### 第四步：清理 TSX 内联颜色
- 将 `SettingsPage.tsx`、`App.tsx` 的内联 `color: '#666'` 等改为类名或变量。
- 优先改为 CSS 类，减少 TSX 中样式逻辑。

### 第五步：统一图表颜色来源
- `DashboardPage.tsx`、`BillingPage.tsx` 中图表 `stroke`、`dot`、网格线、坐标轴颜色统一改为来自变量映射。
- 通过在组件内定义颜色映射常量（读取 CSS 变量）或统一常量文件方式实现，避免重复字面量。

### 第六步：回归与一致性检查
- 全量扫描前端颜色字面量，确保只保留必要品牌色。
- 检查关键页面：Chat、Dashboard、Billing、Settings、Skills、Plugins。
- 运行前端类型检查验证无回归。

## 验收标准
- 前端核心页面不再出现通用颜色硬编码（品牌色除外）。
- 内联颜色显著减少，颜色来源集中到全局变量。
- 图表颜色策略统一，网格与轴颜色一致。
- `npm run typecheck` 通过。
- 视觉上保持当前主题风格，不引入功能变化。

## 风险与应对
- 风险：局部组件依赖旧色值导致视觉细节变化。
  - 应对：按页面逐步替换并即时回归检查。
- 风险：图表库对 CSS 变量读取时机敏感。
  - 应对：采用稳定的颜色映射初始化方案，必要时保留受控常量并集中管理。
