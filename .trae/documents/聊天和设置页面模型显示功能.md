# 聊天和设置页面模型显示功能优化计划

## 需求概述
1. 侧边栏改为分组菜单结构（参考OpenClaw设计）
2. 聊天页面需要显示已配置的模型（添加模型选择器）
3. 设置页面需要显示已保存的模型（新增模型管理功能）
4. 修复设置页面卡片布局问题（当前 max-width: 600px 限制了宽度）

## 实施步骤

### 1. 前端：改造侧边栏为分组菜单结构
- [ ] 修改 `frontend/src/components/Sidebar.tsx`
  - 添加分组菜单结构（参考OpenClaw）：
    - "控制"分组：聊天、概览（仪表盘）、使用情况（计费）
    - "代理"分组：技能、插件、记忆、调试
    - "设置"分组：设置
  - 添加可折叠/展开功能（每个分组右侧有箭头图标）
  - 添加汉堡菜单图标（三横杠）用于收缩侧边栏
  - 添加品牌Logo和名称在顶部
- [ ] 修改 `frontend/src/components/Sidebar.css`
  - 实现分组样式：小标题使用淡灰色，选中的项有红色高亮
  - 添加选中项的样式：浅红色圆角背景 + 红色图标和文字
  - 添加悬停状态的灰色背景
  - 添加可折叠/展开的动画效果
  - 整体风格简洁、扁平化

### 2. 后端：创建模型配置 API
- [ ] 在 `backend/billing/models.py` 中添加模型配置表 `ModelConfiguration`
  - 字段：id, provider, model, is_active, is_default, display_name, description, created_at, updated_at
- [ ] 在 `backend/billing/routers/billing.py` 中添加模型配置相关路由
  - `GET /billing/configurations` - 获取已配置的模型列表
  - `POST /billing/configurations` - 添加模型配置
  - `PUT /billing/configurations/{id}` - 更新模型配置
  - `DELETE /billing/configurations/{id}` - 删除模型配置
  - `PUT /billing/configurations/{id}/set-default` - 设置默认模型
  - `GET /billing/providers` - 获取所有可用的提供商列表
- [ ] 在 `backend/billing/pricing_manager.py` 中添加获取活跃模型的方法

### 3. 前端：创建模型配置服务
- [ ] 在 `frontend/src/services/` 中创建 `modelsApi.ts`
  - 定义 ModelConfiguration 接口
  - 导出模型配置相关 API 方法

### 4. 前端：修改聊天页面
- [ ] 在 `ChatPage.tsx` 中添加模型选择下拉框
  - 在 chat-header 区域添加模型选择器
  - 从 API 获取已配置的模型列表
  - 显示当前选中的模型
  - 支持切换模型
- [ ] 修改 `ChatPage.css`
  - 调整 chat-header 布局，添加模型选择器样式
  - 保持与其他元素的对齐

### 5. 前端：修改设置页面
- [ ] 在 `SettingsPage.tsx` 中添加"模型管理"标签页
  - 显示已配置的模型列表（带勾选框表示是否激活）
  - 添加新模型配置的表单
  - 支持编辑和删除模型配置
  - 支持设置默认模型
- [ ] 修改 `SettingsPage.css`
  - 移除或增大 `settings-content` 的 max-width（改为 100% 或 1200px）
  - 调整模型配置列表的布局样式
  - 添加模型配置卡片的样式

### 6. 前端：模型配置数据同步
- [ ] 在应用初始化时调用 API 获取已配置的模型
- [ ] 将选中的模型保存到 localStorage
- [ ] 发送聊天消息时传递选中的模型信息

## 文件修改清单

### 前端文件

## 预期效果
- 侧边栏采用分组菜单结构，可折叠/展开，带汉堡菜单可收缩
- 选中项有明显红色高亮（浅红色背景 + 红色图标和文字）
- 悬停时有灰色背景效果
- 聊天页面顶部显示模型选择下拉框，用户可以切换不同的模型
- 设置页面的"模型管理"标签页显示所有已配置的模型
- 设置页面的所有内容区域宽度增加，不再受 600px 限制
- 用户可以在设置页面添加、编辑、删除模型配置
