# 代码质量检查清单

## CI 门禁质量检查清单

以下清单在 CI 流水线中自动执行，全部通过后方可合并 PR。

### 1. 静态分析

- [ ] **后端 mypy 零错误**: `mypy backend/ --ignore-missing-imports`
- [ ] **后端 bandit 安全扫描**: `bandit -r backend/ -f json --exit-zero`（P0/P1 不允许新增）
- [ ] **前端 TypeScript 零错误**: `tsc --noEmit`
- [ ] **前端 ESLint 零错误**: `eslint "src/**/*.{ts,tsx}" --max-warnings=0`

### 2. 单元测试

- [ ] **后端 pytest 全部通过**: `pytest tests/ -x --tb=short`
- [ ] **前端 vitest 全部通过**: `vitest run --reporter=verbose`
- [ ] **测试通过率 >= 95%**（允许已知的 skip 测试）

### 3. 构建验证

- [ ] **前端构建成功**: `npm run build`
- [ ] **无构建警告**

### 4. 安全扫描

- [ ] **依赖安全审计**: `npm audit --audit-level=high`（前端），`pip audit`（后端）
- [ ] **无已知漏洞依赖**

### 5. 代码规范

- [ ] **无 console.log 残留**: 使用 eslint `no-console` 规则
- [ ] **无 TODO/FIXME 未处理**: 使用 eslint 或 grep 检查
- [ ] **无调试代码残留**

## Code Review 检查清单

以下清单在每次 Code Review 时由 Reviewer 逐项检查。

### 通用检查

- [ ] **功能正确性**: 代码实现了 PR 描述的功能
- [ ] **无回归风险**: 变更不会破坏现有功能
- [ ] **测试覆盖**: 新增/修改的代码有对应的测试用例
- [ ] **测试通过**: 所有测试（包括已有测试）通过

### 异常处理

- [ ] **异常类型具体化**: 不使用宽泛的 `except Exception`，使用具体异常类型
- [ ] **禁止空 except**: 不存在 `try/except/pass` 模式
- [ ] **异常传播**: 关键路径的错误正确传播到上层，不静默吞异常
- [ ] **资源释放**: 文件、连接、流式 response 等资源在使用后正确释放
- [ ] **超时控制**: 外部调用（LLM、数据库、网络）设置了超时

### 输入验证

- [ ] **Schema 校验**: 所有外部输入有 Pydantic/zod Schema 校验
- [ ] **长度限制**: 字符串类型字段有最大长度限制
- [ ] **文件上传安全**: 文件大小校验、类型白名单、路径穿越防护
- [ ] **命令执行安全**: 参数转义、沙箱隔离、路径白名单

### 日志记录

- [ ] **上下文完整**: 日志包含 `request_id`、`user_id` 等关联信息
- [ ] **无敏感信息**: 日志中不包含 token、密钥、个人信息
- [ ] **日志级别正确**: ERROR 用于未预期异常，WARNING 用于预期异常

### 并发安全

- [ ] **全局状态保护**: 模块级可变状态使用锁保护
- [ ] **异步安全**: 事件循环中无阻塞调用
- [ ] **组件卸载安全**: 组件卸载时取消请求、清理定时器、关闭连接

### 类型安全

- [ ] **无 any 类型**: TypeScript 代码不使用 `any`（可用 `unknown` 替代）
- [ ] **函数签名完整**: Python 函数有完整类型标注，TypeScript 函数参数和返回值有类型
- [ ] **None 安全**: Python 代码访问属性前检查 None

### 性能

- [ ] **ORM 查询**: 使用 `joinedload`/`selectinload` 避免 N+1
- [ ] **大数据量分页**: 列表查询使用分页
- [ ] **不必要的重渲染**: React 组件使用 `React.memo` 或 `useMemo` 优化

### 代码风格

- [ ] **命名规范**: 遵循项目的命名约定
- [ ] **无重复代码**: 提取公共逻辑到可复用函数/组件
- [ ] **导入顺序**: 标准库 -> 第三方库 -> 项目内部
- [ ] **无死代码**: 移除未使用的导入、变量、函数

### 安全专项

- [ ] **SSRF 防护**: URL 代理端点有域名白名单
- [ ] **路径遍历防护**: 文件操作使用 `Path.resolve()` + `relative_to()` 约束
- [ ] **SQL 注入防护**: 使用 ORM 参数化查询，不使用拼接 SQL
- [ ] **XSS 防护**: 用户输入渲染使用安全的转义/净化方式

### 变更影响

- [ ] **API 向后兼容**: 已有 API 变更需提供兼容层
- [ ] **数据库迁移**: Schema 变更有对应的迁移脚本
- [ ] **配置变更**: 新增配置有默认值和文档说明
- [ ] **前端国际化**: 新增文案添加到国际化文件
