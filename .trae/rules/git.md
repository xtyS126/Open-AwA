在编辑完文件后，执行 `git add .` 命令将所有修改的文件添加到暂存区。
在执行 `git commit -m "描述性提交信息"` 命令提交修改，其中"描述性提交信息"应简洁明了地说明本次提交的内容。
应当在本次修改结束后审核代码，确保没有引入错误或不兼容的变更。
应当在提交前检查代码是否符合项目的编码规范和风格。
应当在提交前运行测试，确保所有测试通过。
应当在提交前更新项目的文档，包括 README.md、API 文档等。
应当在提交前检查项目的依赖项是否更新，确保与其他模块兼容。
应当排除任何与项目功能无关的文件，如临时文件、缓存文件等。
应当在本次修改完成后审核代码，然后提交本地仓库。
