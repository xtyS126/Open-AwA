# Checklist

## 性能与渲染优化
- [x] 后台节流：当 `document.hidden === true` 时，SSE 的数据块不再导致每一次接收都触发 React 渲染，而是累加至缓冲区。
- [x] 后台更新降频：即便在后台累加数据，也会根据固定时间间隔（如 1000 毫秒）执行一次状态更新以防内存失控。
- [x] 取消不必要的后台操作：在后台时不触发 `messagesEndRef.current?.scrollIntoView` 等 DOM 滚动更新动画。

## 恢复与状态刷新
- [x] 页面恢复：监听 `visibilitychange` 事件，当页面回到前台 (`document.hidden === false`) 时，立即将所有在缓冲区中的文本及思维链推送更新。
- [x] 恢复后滚动：页面回到前台后，立即自动平滑滚动到最新内容底部。
- [x] 流程结束兜底：流式传输无论正常结束还是抛出错误结束，都能将缓冲区残留的内容一次性清空并保存，防止丢字。

## 质量验证
- [x] ESLint 和 TypeScript 类型检查没有因为使用 `useRef` 及缓冲区而产生错误。
- [x] 测试验证确认了长时间流式输出时切换后台，返回后不会发生任何模型结果被截断的情况。
