# 系统优化 Q2 2026 Spec

## Why

当前系统存在四个需要解决的优化点：微信自动回复功能偶尔失效需要排查修复；MCP 配置分散在多处需要统一迁移；技能创建页面 UI 和流程需要现代化改造；聊天页面模型选择入口造成用户困惑需要精简交互路径。

## What Changes

### 模块一：微信自动回复功能修复
- 排查并修复绑定状态检查、回调地址验证、事件推送日志记录问题
- 规范化自动回复接口响应格式（XML/JSON）以符合微信官方要求
- 增加前端点击事件埋点，确保用户操作能够正确触发后台回复逻辑
- 输出包含根因分析、修复代码、回归测试的完整修复报告

### 模块二：MCP 配置迁移
- 将现有 MCP 配置统一迁移到代理服务配置中心
- 确保配置热更新能够正确生效
- 验证迁移后依赖 MCP 的功能（技能创建、消息路由）无降级
- 提供配置版本快照与回滚方案

### 模块三：技能创建页面重构
- UI 美化：统一视觉规范，优化信息密度与留白比例
- 流程精简：删除冗余入口，合并相似操作，步骤控制在 3 步以内
- 功能对齐：与产品文档保持一致，确保字段、权限、文案、校验规则准确
- 输出前后对比截图、交互走查报告、可访问性评分达 90 分以上

### 模块四：聊天页模型选择入口移除
- 从聊天页面顶部彻底移除模型选择器及其相关状态管理代码
- 在"设置-通用设置"新增"主模型选择"菜单，保留原有模型列表与默认值逻辑
- 确保切换主模型后全局生效，已开启对话实时同步新模型
- 提供埋点数据证明迁移后用户路径缩短、误操作率下降

## Impact

- Affected specs: 微信集成模块、设置页、聊天页、技能管理模块、MCP 集成
- Affected code:
  - `backend/api/routes/weixin.py` - 微信自动回复接口
  - `backend/api/routes/skills.py` - 技能与 MCP 配置接口
  - `backend/config/` - MCP 配置中心
  - `frontend/src/features/chat/ChatPage.tsx` - 聊天页模型选择移除
  - `frontend/src/features/settings/SettingsPage.tsx` - 通用设置与主模型选择
  - `frontend/src/features/skills/` - 技能创建页面重构
  - `frontend/src/shared/api/api.ts` - API 类型定义

## ADDED Requirements

### Requirement: 微信自动回复接口响应格式规范化
系统 SHALL 确保微信自动回复接口返回格式符合微信官方要求，支持 XML 和 JSON 两种格式自动适配。

#### Scenario: 微信服务器回调触发自动回复
- **WHEN** 微信服务器向回调地址推送用户消息事件
- **THEN** 系统正确解析消息内容并调用 AI 生成回复
- **THEN** 回复内容以微信要求格式返回（XML 格式或 JSON 格式根据请求 Accept 头决定）

### Requirement: 微信事件推送日志完整记录
系统 SHALL 记录所有微信服务器事件推送的详细日志，便于问题排查和审计。

#### Scenario: 查看微信事件推送日志
- **WHEN** 用户或管理员需要排查微信消息处理问题
- **THEN** 系统提供完整的事件推送日志，包含时间戳、消息类型、内容、处理结果

### Requirement: MCP 配置热更新生效
系统 SHALL 在 MCP 配置更新后无需重启服务即可生效。

#### Scenario: MCP 配置热更新
- **WHEN** 管理员在配置中心修改 MCP 配置
- **THEN** 相关功能模块在合理时间内（通常不超过 30 秒）感知到配置变更并应用新配置

### Requirement: 技能创建页面步骤简化
系统 SHALL 将技能创建流程控制在 3 步以内完成。

#### Scenario: 用户创建新技能
- **WHEN** 用户进入技能创建页面
- **THEN** 用户可以在 3 步内完成：基本信息填写 -> 核心配置 -> 保存确认

### Requirement: 主模型选择全局生效
系统 SHALL 在"设置-通用设置"中选择的"主模型"对所有对话生效。

#### Scenario: 全局模型切换
- **WHEN** 用户在设置中切换主模型
- **THEN** 新开启的对话使用新模型
- **THEN** 已开启的对话实时同步为新模型，无需刷新页面

## MODIFIED Requirements

### Requirement: 聊天页模型选择交互简化
系统 SHALL 移除聊天页面的模型选择器，用户如需切换模型应通过设置页面操作。

#### Scenario: 发起新对话使用默认模型
- **WHEN** 用户发起新对话
- **THEN** 系统自动使用"设置-通用设置"中指定的主模型
- **THEN** 用户无需在聊天页选择模型

### Requirement: 技能创建 UI 现代化
系统 SHALL 重构技能创建页面，采用统一的视觉规范和交互模式。

#### Scenario: 技能创建页面可访问性
- **WHEN** 用户访问技能创建页面
- **THEN** 页面符合 WCAG 2.1 AA 标准，可访问性评分达到 90 分以上
- **THEN** 页面在桌面端和移动端均有良好体验

## REMOVED Requirements

### Requirement: 聊天页模型选择器
**Reason**: 当前模型选择入口分散在聊天页和设置页，造成用户体验困惑和误操作。统一在设置页管理可降低复杂度。
**Migration**: 原聊天页模型选择功能迁移至"设置-通用设置-主模型选择"，功能完整保留。

### Requirement: 分散的 MCP 配置文件
**Reason**: MCP 配置分散在不同位置导致维护困难和热更新失效风险。
**Migration**: 统一迁移至代理服务配置中心集中管理。

## Technical Notes

### 微信自动回复排查要点
- 绑定状态：`binding_status`、`confirmed` 字段是否正确持久化
- 回调地址：`wx_callback_url` 是否可达，是否完成 URL 验证
- 事件日志：`get_updates_buf` 是否正确记录消息上下文
- 响应格式：`ToUserName`、`FromUserName` 字段对应关系是否正确

### MCP 配置迁移策略
- 使用配置中心统一存储 MCP 配置
- 配置变更时触发热更新事件
- 保留配置版本快照支持回滚
- 迁移前进行配置备份

### 技能创建页面重构原则
- 采用 design token 统一设计变量
- 使用渐进式披露（Progressive Disclosure）减少初始表单字段
- 表单验证采用实时校验 + 提交前完整校验双重保障
- 支持键盘导航和屏幕阅读器
