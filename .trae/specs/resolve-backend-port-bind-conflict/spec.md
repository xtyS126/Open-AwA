# 修复后端端口绑定冲突 Spec

## Why
当前在 Windows 环境执行 `python main.py` 时，后端会在完成应用启动后因端口 8000 已被占用而抛出 `WinError 10048`，导致服务立即退出。该问题会阻断本地开发与联调，需要让启动流程在端口冲突场景下提供可恢复、可诊断的行为。

## What Changes
- 为后端启动流程补充端口占用检测与绑定失败处理能力
- 在默认端口被占用时输出清晰、可操作的错误信息，提示冲突端口与处理建议
- 评估并实现安全的端口回退策略或显式终止策略，保证行为一致且可预测
- 保持现有应用初始化、生命周期日志与路由注册逻辑不被破坏
- 为启动入口补充针对端口冲突场景的验证覆盖

## Impact
- Affected specs: 后端服务启动、开发环境运行、启动错误诊断
- Affected code: `backend/main.py`、启动入口脚本、配置读取逻辑、相关测试代码

## ADDED Requirements
### Requirement: 端口冲突诊断
系统 SHALL 在启动后端服务前或绑定端口失败时，识别端口占用冲突并输出明确诊断信息。

#### Scenario: 默认端口已被占用
- **WHEN** 用户执行 `python main.py` 且默认监听端口已被其他进程占用
- **THEN** 系统应明确指出冲突端口与错误原因
- **THEN** 系统应给出可执行的处理建议，例如更换端口或释放占用进程
- **THEN** 系统不得仅输出底层异常而缺少面向用户的说明

### Requirement: 启动行为可预测
系统 SHALL 在端口冲突场景下采用单一且稳定的处理策略，避免出现应用已部分启动但最终异常退出且无明确指引的情况。

#### Scenario: 启动流程遇到绑定失败
- **WHEN** 应用完成初始化后发生端口绑定失败
- **THEN** 系统应按约定策略处理该失败
- **THEN** 系统应保证日志、退出状态与提示信息相互一致

## MODIFIED Requirements
### Requirement: 本地开发启动流程
系统在本地开发环境下启动后端服务时，除完成数据库初始化、计费表初始化与日志初始化外，还必须对监听端口可用性负责，并在端口不可用时提供可恢复的启动反馈。

## REMOVED Requirements
### Requirement: 无
**Reason**: 本次变更以增强启动健壮性为主，不移除现有功能。
**Migration**: 无需迁移。
