# Checklist

## 后端接口与协议层
- [x] `ChatMessage` 入参已成功支持 `mode` 参数。
- [x] `/chat` 接口能够正确区分 `mode=stream` 与 `mode=direct`，并分别返回 SSE (`text/event-stream`) 响应或标准 JSON 响应。
- [x] `model_service.py` 已具备通过 `httpx` 建立稳定流式请求的能力，并正确处理请求级别的错误与重试。
- [x] Agent 执行层能正常提取并向外抛出大模型的 `reasoning_content`（思维链片段）和 `content`（正文片段）。

## 前端交互与渲染
- [x] 聊天页面（ChatPage）成功渲染可切换的“输出模式”控件（默认流式传输）。
- [x] 切换模式后，配置已持久化至 `localStorage` 且下一次聊天请求即刻生效（无需刷新页面）。
- [x] 流式模式下，客户端正确发送带有 `mode=stream` 的请求，并能够正常解析后端的 SSE 响应流，实现逐字实时渲染。
- [x] 若模型响应中含有思维链数据，界面上能在独立且易区分的区域（如折叠面板或特殊背景色）实时渲染思维链内容。
- [x] 两种模式下的交互（Loading 动画、Error 提示、重试按钮逻辑）均保持高度一致和稳定。
