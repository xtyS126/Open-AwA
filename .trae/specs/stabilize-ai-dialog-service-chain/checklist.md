# Checklist

## 根因诊断
- [ ] 已定位 `Error: Model service request failed` 的主要触发路径
- [ ] 已覆盖网络超时、认证失效、请求格式错误、限流与后端异常的分类日志与堆栈
- [ ] 前后端日志已可通过 request_id 关联同一条失败链路

## 前端兼容性
- [ ] SSE / WebSocket 流式响应能够按序解析，不丢包、不截断
- [ ] 思维链分段缺失、乱序或校验失败时有明确降级处理
- [ ] 工具调用返回新增字段时前端可忽略未知字段
- [ ] 工具调用返回缺失字段时前端提供默认值并提示兼容模式

## 后端兼容性
- [ ] 模型网关已按 `X-Client-Ver` 返回兼容格式
- [ ] 思维链分段输出包含序号与校验和
- [ ] 工具调用具备 `idempotency-key` 与超时重试机制
- [ ] 副作用型工具在重试场景下不会被重复执行

## 错误处理与观测
- [ ] 标准错误码与映射表已定义并贯通前后端
- [ ] 前端可按错误码展示文案
- [ ] 后端 Prometheus 指标已暴露关键稳定性数据
- [ ] Grafana 看板所需指标字段已明确并可验证

## 测试与交付
- [ ] 单元测试覆盖工具调用场景与思维链输出格式
- [ ] 集成测试覆盖 5s 延迟、502/503、chunked 截断等异常
- [ ] 已形成压测结果，验证 200 并发 10 分钟下的成功率、P99 与内存表现
- [ ] 已整理代码变更说明、部署文档、回滚步骤、测试报告与性能对比截图清单
